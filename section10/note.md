# 유틸리티 타입

> 제네릭, 맵드 타입, 조건부 타입 등의 타입 조작 기능을 이용해 실무에서 자주 사용되는 타입을 미리 만들어 놓은 것

- 유틸리티 타입은 복잡한 타입 로직을 미리 정의하여 타입 코드의 재활용성을 높이고 정의를 간결함
- Partial은 모든 속성을 ?를 붙여 선택적으로, Required는 반대로 필수, Readonly는 읽기 전용, Pick은 특정 속성만
- `Record<K, V>`는 키가 K 타입, 값이 V 타입인 객체를 정의, 인덱스 시그니처와 비슷하지만 키 타입을 더 구체적으로 제한한다.  
  다른 타입은 객체의 형태를 바꾸거나 함수 반환 타입을 추출
