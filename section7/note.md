- 비동기 처리를 이용하면 오래 걸리는 작업이 종료될 때까지 기다리지 않고 다음 작업을 수행하는 등 유연한 프로그래밍이 가능
- 제네릭은 다양한 타입을 다루는 코드를 작성할 때, 컴파일 시 타입 오류를 잡아내면서도 유연한을 유지.
- 제네릭 함수는 호출할 때 전달된 인자의 타입을 보고 TypeScript 컴파일러가 자동으로 타입 변수 T의 타입을 추론. 명시적 지정도 가능하지만 주로 추론
- 제네릭 타입 변수 `T`에 `extends SomeType`와 같은 제약을 거는 이유: `extends` 키워드를 사용하면 타입 변수 `T`의 타입을 특정 구조나 속성을 가진 타입으로 제한
- `Promise<T>`에서 `<T>`는 Promise가 성공적으로 완료되었을 때 `.then()`메서드로 넘겨받을 값의 타입을 명확히 정의. 비동기 결과의 타입 안정성을 확보
- 제네릭 함수는 호출 시 인자로 타입을 추론하는 경우가 많지만, 제네릭 인터페이스나 타입 별칭은 사용할 때 `<MyType>`처럼 명시적으로 타입을 지정해주는 방식이 일반적
