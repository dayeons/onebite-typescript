- 인터페이스: 타입에 이름을 지어주는 또 다른 문법, 상호간에 약속된 규칙  
  객체의 구조를 정의하는데 특화된 문법 (상속, 합침 등의 특수한 기능을 제공함)

- 인터페이스는 객체 타입 정의에 특화되어 있으며, 타입 별칭과는 다르게 상속을 통해 속성을 물려받거나 동일 이름으로 여러 번 선언 시 자동으로 합쳐지는 기능
- 메소드 오버로딩은 이름은 같지만 시그니처가 다른 메소드를 여러 개 정의하는 것, 인터페이스에서는 호출 시그니처를 사용해야만 이러한 형태가 정의
- 인터페이스 확장은 다른 인터페이스의 속성을 물려받아 새로운 인터페이스를 정의. 이를 통해 여러 관련 타입에서 공통된 속성을 반복해서 정의할 필요가 없다
- 타입 별칭과 달리 인터페이스는 동일 이름으로 여러 번 선언해도 오류 없이 정의된 속성들이 모두 합쳐져 하나의 타입처럼 동작 = 선언 합침
- 선언 합침은 이미 정의된 타입(라이브러리 타입)에 새로운 속성이나 메소드를 추가하여 타입 정의를 확장할 때 유용 = 모듈 보강
