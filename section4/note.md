- 함수 오버로딩: 함수를 매개변수의 개수나 타입에 따라 여러가지 버전으로 정의하는 방법

- 함수 매개변수에 `?` 기호를 붙여 선택적 매개변수로 만들었을 때, 해당 매개변수의 타입은 어떻게 추론될까요?  
  선언된 타입 또는 `undefined` 의 유니언 타입  
  선택적 매개변수는 값을 제공하지 않을 수 있으므로, 원래 선언된 타입과 `undefined`가 합쳐진 유니언 타입으로 추론
- 변수가 함수처럼 호출되면서 동시에 객체의 속성도 가질 수 있는 '하이브리드 타입'을 정의할 때 사용하는 문법: 호출 시그니처  
  호출 시그니처는 함수 타입을 객체 타입 문법 안에 정의하여, 변수가 함수와 객체 속성을 동시에 가질 수 있는 하이브리드 타입
- 할당하는 쪽 매개변수 타입이 더 넓을 때: 매개변수 개수가 같은 두 함수 타입의 호환성 검사 시, 매개변수 타입에 대해 일반적으로 허용되는 할당 방향  
  함수 타입 호환성에서 매개변수 타입은 공변성이 아닌 반공변성을 따릅니다. 즉, 할당하는 쪽(소스)의 매개변수 타입이 할당받는 쪽(대상)보다 더 넓을 때(다운캐스팅) 호환
- 오버로드 시그니처는 실제 구현부 없이 함수가 어떤 형태의 매개변수로 호출될 수 있는지 타입 레벨에서 미리 알려주는 역할, 이를 통해 컴파일러가 호출의 유효성 검사
- `매개변수명 is 타입` 은 사용자 정의 타입 가드 문법. 이 함수가 `true`를 반환하면 Typescript는 해당 매개변수가 지정된 타입임을 인지하고 타입 좁히기를 수행.
