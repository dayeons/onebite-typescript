- 클래스는 동일한 속성과 기능을 가진 여러 객체를 생성할 때 사용하는 '틀' 또는 '설계도'역할. 객체 리터럴의 코드 중복 문제 해결
- 상속은 부모 클래스의 기능을 자식 클래스가 이어받아 코드의 재사용성을 높이는 개념 `extends`
- `private` 오직 해당 클래스 자신 안에서만 접근 허용. 인스턴스나 자식 클래스에서는 접근할 수 없어 가장 제한적
- 인터페이스의 주요 역할: 클래스가 구현해야 할 구조(속성, 메서드, 시그니처)를 정의하는 설계도 역할 `implements`
- TypeScript에서 클래스는 객체 생성자일 뿐만 아니라, 그 자체로 타입으로 활용. 이는 코드에서 변수나 함수 인자의 타입을 명확히 하여 안정성을 높이는 역할
